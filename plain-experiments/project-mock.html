GAME DESIGN

King's Command

A turn-based tactics game where players command autonomous AI soldiers through natural language orders. Exploring the boundary between strategic intent and emergent behaviour.

The premise

What if you could command units in a tactics game the way you'd brief a team in real life? "Flank from the left, hold fire until they're in range, retreat if outnumbered." The game became a vehicle for exploring how LLMs interpret tactical language.

Strategy emerges from the gap between what you say and what the system understands.
order_parser.py — extracting intent from natural language
# First pass: chunk the order into atomic actions
def parse_order(order: str) -> list[Action]:
    intent = llm.extract(order, schema=ActionSchema)
    return [Action(**a) for a in intent.actions]
Process note
Early iterations had the LLM generating moves directly. That felt like a black box. We switched to intent extraction + a deterministic execution layer so players could debug "why did it do that?"

Project updates

2024-03-01

First playable

Got the basic loop working. Players type orders, units execute. The "retreat if outnumbered" logic is still brittle — need to define a clearer threshold system.

2024-02-15

Switched to intent extraction

After a few playtests, direct LLM-to-move was too opaque. Refactored to extract structured intent first, then run a deterministic executor. Much easier to reason about.

2024-01-20

Project kickoff

Started exploring natural language as a command interface for tactics games. Initial prototype used a simple keyword parser — quickly hit its limits.
